The atnet library and software

Up to SDK version 4.7.1, the atnet64 software was released as a precompiled executable, which was able to communicate with the fusionTrack bootloader (aka factory firmware). Using atnet from another software was therefore not very convenient. It was therefore decided to rewrite this software, transforming it into a shared library, which might be directly used in any application.

The API consists of a set of function, POD structures and callbacks. For instance, detecting connected fusionTracks reads:

#include <atnetInterface.h>
#include <iostream>
#include <memory>
#include <new>
#include <string>
using namespace std;
void displayCommandOutput( const char* msg, void* )
{
cout << msg << endl;
}
int main( int argc, char* argv[] )
{
const size_t errorBufferSize( 1024u );
unique_ptr< char[] > errorBuffer( new (nothrow) char[ errorBufferSize ] );
string configFile( "" ); // path to a JSON configuration file for
atnetLib libraryHandle( atnetInit( configFile.c_str(), errorBufferSize, errorBuffer.get() ) );
if ( libraryHandle == nullptr )
{
cerr << errorBuffer.get() << endl;
return 1;
}
atnetError status( atnetExecuteCommand( libraryHandle, "scan", nullptr, nullptr, nullptr, nullptr ) );
if ( status != atnetError::Ok )
{
if ( atnetGetError( handle, errorBufferSize, errorBuffer.get() ) == atnetError::Ok )
{
cerr << errorBuffer.get() << endl;
}
else
{
cerr << "Unknown error scanning for devices" << endl;
}
atnetClose( &libraryHandle );
return 1;
}
status = atnetExecuteCommand( libraryHandle, "list", displayCommandOutput, nullptr, nullptr, nullptr );
if ( status != atnetError::Ok )
{
if ( atnetGetError( handle, errorBufferSize, errorBuffer.get() ) == atnetError::Ok )
{
cerr << errorBuffer.get() << endl;
}
else
{
cerr << "Unknown error listing discovered devices" << endl;
}
atnetClose( &libraryHandle );
return 1;
}
atnetClose( &libraryHandle );
return 0;
}

The atnet API is documented in The atnet library.

Available commands

The commands provided by the atnet library are called via a command string, consisting of the command name and its arguments. The commands are read and processed by the atnetExecuteCommand function. The output of the command is passed to a callback (which signature is given by the processOutputCallback typdef) as a const char*. For each answerwing device, the answer consists of "{serial_number:output_args;}", each output argument identified by its name. Consecutive output arguments are separated by ;. If the command was sent and executed on several devices, their answers are concatenated in the same output string, with a new line character ( \n) between the answer of two devices.

This page present the atnet commands as if they were C++ functions. The input arguments can therefore be properly documented. The output arguments (when there are some) are always std::tuple, and only represent the answer of a single device. What the atnet library gives as output is always a string, in which the answers of all devices are concatenated. For instance, the list command is documented to return a std::tuple containing a uint64 value only. If two devices were detected, the retrieved string will be

"{serial_number_1:sn=serial_number;}
{serial_number_2:sn=serial_number;}"

ftkLibrary ftkInit ( )

Function initialising the library.

This function initialises the library and creates the handle needed by the other library functions. It must therefore be called before any other function from the library.

Returns
the library handle or nullptr if an error occurred.
1 // Initialize and close library
2 
3 ftkLibrary handle( ftkInit() );
4 if ( handle == nullptr )
5 {
6  ERROR( "Cannot open library" );
7 }
8 
9 // ...
10 
11 if ( ftkClose( &handle ) != ftkError::FTK_OK )
12 {
13  ERROR( "Cannot close library" );
14 }

ftkLibrary ftkInitExt ( const char *  fileName,
ftkBuffer *  errs 
)

`Extended' initialisation of the library.

This function allows to customise the library default initialisation. It currently provides support for changing the IP address of the device it connects to. Please note the device port default value cannot be changed. The configuration file uses JSON format, with the following syntax: 

{
  "network" :
  {
    "version": 1,
    "interfaces": [
      {
        "address": "172.17.10.7",
        "port": 3509
      }
    ]
  }
}

If several interfaces are defined, then the SDK will allow to connect to multiple fusionTrack devices. This means the ftkEnumerateDevices will be able to detect up to one device per defined interface. Each function taking a serial number as input argument will behave correctly, i.e. will interact with the device with the given serial number. When using multiple fusionTrack with the same SDK, it is recommended to set them to use distinct subnetworks (i.e. only the two first bytes of the IP address match). The function also allows the caller to get information about initialisation errors, as shown on the following snippet:

1 ftkBuffer buffer;
2 ftkLibrary lib( ftkInitExt( "configuration.json", &buffer ) );
3 
4 if ( lib == nullptr )
5 {
6  cerr << buffer.data << endl;
7  cerr << "Cannot initialise library" << endl;
8  return 1;
9 }
Parameters
[in]fileNamename of the JSON file from which the configuration will be read.
[out]errspointer on the buffer containing potential errors.
Returns
the library handle or nullptr if an error occurred.
ftkError ftkClose ( ftkLibrary *  ptr)

Close the library.

This function destroys the library handle and frees the resources. Any call to a library function after this will fail.

Parameters
[in]ptrpointer an initialised library handle
Return values
ftkError::FTK_OKif the library could be closed successfully,
ftkError::FTK_ERR_INV_PTRif the ptr is 0 or if the library handle pointed by ptr has already been closed.

ftkError ftkGetLastErrorString ( ftkLibrary  lib,
size_t  strSize,
char  str[] 
)

Getter for the last error.

This function allows to retrieve the last error (i.e. the error from the previous call to another SDK function). The error is formatted as an XML string: 

<ftkError>
  <errors> ... </errors>
  <warnings> ... </warnings>
  <messages> ... </messages>
</ftkError>

The errors tag contains the error codes and strings, the warning tag contains the warning codes and string. The messages tag contains optional extra messages.

Parameters
[in]libinitialised library handle.
[in]strSizesize of the string passed as argument.
[in,out]strallocated character array containing the output XML.
Return values
ftkError::FTK_OKif the retrieving could be done successfully,
ftkError::FTK_ERR_INV_PTRif lib or str is 0.

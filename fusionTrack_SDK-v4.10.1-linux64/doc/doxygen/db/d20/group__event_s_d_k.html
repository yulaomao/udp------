struct ftkEvent

Structure holding an event as sent by the tracker.

This structure implements the device-oriented data structure of an event.

Events are generated and sent by the tracker itself. Some are handled directly by the driver itself, others are forwarded to the user. In order to get those event in a custom made program, the ftkReadEvent() function is used.

  4. camera 1, sensor 1;

fetTempV3 

Temperature sending, version 3.

Deprecated:
This event is not anymore forwarded to the user.

The payload contains the 8 float32 values for the following temperatures:

  1. camera 0, sensor 0;
  2. camera 0, sensor 1;
  3. camera 1, sensor 0;
  4. camera 1, sensor 1;
  5. IR board 0;
  6. IR board 1;
  7. main board;
  8. power supply board; followed by:
  9. one byte indicating the status of the fan readings;
  10. 1 byte for the fan 0 input in percent;
  11. 2 bytes for the fan 0 speed;
  12. 1 byte for the fan 1 input in percent;
  13. 2 bytes for the fan 1 speed;
  14. 1 reserved byte (unused).

This lists the possible returned error by the PTP protocol.

Warning
The error is actually 6 bits wide.
Enumerator
Ok 

No Error.

NoMasterDetected 

No master detected.

NetworkTimeoutError 

Too long time waiting packets in the switch or in the master.

AnnounceWrongIntervalError 

Wrong interval between Announce message detected, please check if the setup value of `Log Announce Interval' is correct.

SyncWrongInternalError 

Wrong interval between Sync message detected, please check if “Log Sync Interval” is correct.

TooMuchMastersError 

There are too much foreign master (more than five) communicating in the network.

SyncMessageTimeout 

Sync message timeout (no sync message received since detection of the parent Master).

NoSyncFromMaster 

No sync message detected from the parent Master (or in wrong order).

SyncMessageError 

Problem encounter with the sync messages (check length and sequence ID).

NoFollowUp 

No follow-up message detected but the two-step flag was set.

FollowUpSyntaxError 

Problem encounter with the follow-up messages (check length and sequence ID).

NoDelayAnswer 

No delay-resp message detected from the parent Master (or in wrong order).

DelayAnswerSyntaxError 

Problem encounter with the delay-resp messages (check length and sequence ID).

UnstableSynchronisationError 

The Precision Time Protocol can be applied but the synchronization appears to be unstable.

WrongDomain 

Wrong PTP domain (check if the Domain Number has been correctly set).

UnsupportedVersion 

The PTP version is not supported (only v2.0 supported).

UnsupportedFlags 

Messages flags do not correspond to fTk PTP usage.

Please note the alternate master, unicast and profile specific flags should be zero as not supported.

UnknownError 

Unknown error, used if the error ID cannot be transformed in a ftkErrorPTP.

LastError 

Last error (currently max size for Error ID is 6 bits).

enum ftkPortStatePTP : uint8
strong

Enumeration representing the PTP module FSM state.

Enumerator
Initialising 

State to initialize the data sets (instance not sync, no PTP communication).

Faulty 

State to indicate failure (instance not sync, no PTP communication).

Listening 

State in which the instance listen to Announce messages and apply the BMCA (instance not sync, PTP communication only for reception).

Uncalibrate 

State used when a new master is detected. Initialize the synchronisation servos and getting synchronised (instance not sync, slave PTP communication active).

Slave 

State in which the instance is synchronised to the selected master port (instance sync, slave PTP communication active)

UnknownState 

Unknown state, used if the convertion could not be performed.

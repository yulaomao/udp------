Detailed Description

The atnet library is the new way (starting from SDK 4.8.1) to communicate with the fusionTrack bootloader (aka factory firmware), for maintenance purpose.

A very short example is given on the The atnet library and software page.

typedef void processProgressOutputCallback(const char *message, int oldPercentage, int newPercentage, void *user)

Definition of the callback function for the atnetExecuteCommand function.

This callback signature is used if the use of the atnetExecuteCommand function want to be notified of the progress of some lenghty commands.

The first call to this callback will have as parameters the message, -1 as the old percentage and 0 as the new one. The last call (not depending whether the counter reached 100% or not) will have 0 as old value and -1 as the new one.

Parameters
[in]messagemessage of the command (for command using this feature).
[in]oldPercentageold value of the progress (in percent) of the command (for command using this feature).
[in]newPercentagenew value of the progress (in percent) of the command (for command using this feature).
[in,out]userpointer on user data.

enum atnetError : int32

Status codes for the atnet library.

Enumerator
WarAlreadyProcessingCommand 

A command is already being processed.

Ok 

No errors.

ErrInvPointer 

Provided pointer is nullptr.

ErrInvSerial 

Provided serial number does not belong to the list of enumerated devices.

ErrEmptyCommand 

The provided command is empty.

ErrUnknownCommand 

The provided command does not exist.

ErrInternal 

Internal error, should not happen.

ErrArgumentQuote 

A provided argument was ill-formed (bad quoting).

An error in the quotation marks was detected. For instance a missing " or a missing ' was detected.

ErrInputArgument 

Error when handling the input arguments.

Either a mandatory argument was found missing or its value does not correspond to the argument type.

ErrCannotProcessCommand 

The command could not be prepared.

This is for instance issued if an error occurred when preparing the UDP packet.

ErrRunningCommand 

The command could not be run.

This error can be caused by the packet not being sent, or the answer not received or an error was reported by the fusionTrack.

ErrCannotExtractOutput 

The command could be run but the output could not be read.

This error occurs when the command is successfully sent to the fusionTrack, successfully processed by it, but output arguemnts couldn't be extracted from the received packet.

ErrBufferTooSmall 

The provided buffer is too small.

Function trying to execute the wanted function.

This function tries to extract the wanted function from the args arguments:

  • the function name is extracted and checked to exist;
  • the arguments given to the function are extracted;
  • the arguments are checked to be compatible with the wanted function;
  • the function is executed.
Parameters
[in]libinitialised library handle.
[in]argsarguments string provided to the function.
[in]funccallback function, called for each output of the command.
[in,out]userOutputpointer on user-data, simply forwarded to the func function.
[in]progresscallback function, called for each iteration of some complex commands.
[in,out]userProgresspointer on user-data, simply forwarded to the progress function.
Return values
atnetError::Okif the command could be successfully executed,
atnetError::ErrInvPointerif lib or func is nullptr,
atnetError::ErrEmptyCommandif no commands are provided,
atnetError::ErrUnknownCommandif the provided command does not exist,
atnetError::ErrArgumentQuoteif the provided command is ill-formed because of mismatching number of single or double quotes for instance,
atnetError::ErrInputArgumentif any of the input arguments was given a n invalid value or if a mandatory argument was not provided,
atnetError::ErrCannotProcessCommandif the internal data could not be written before trying to send the command to the fusionTrack,
atnetError::ErrRunningCommandif the command could not be sent to the fusionTrack (some pointer invalid, could not determine if several calls are needed, etc. or if the command execution on the fusionTrack returned an error,
atnetError::ErrCannotExtractOutputif the extraction of the command output could not be performed,
atnetError::ErrInternalif an allocation error occurred,
atnetError::WarAlreadyProcessingCommandif a command is already being executed.
